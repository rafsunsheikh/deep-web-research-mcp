### Instructions for Codex: Build an Autonomous Web Search MCP Server

**Context & Goal:**
You are acting as an expert Python engineer. Your task is to build a complete Model Context Protocol (MCP) server using the `FastMCP` framework (from the official `mcp` Python SDK). The server will provide a comprehensive "deep research" tool for a local LLM. It must search the web via a local SearXNG instance, extract URLs and metadata, scrape the page content and backlinks using Trafilatura, index and rank the extracted text based on the search query, and return the highly relevant, structured results to the LLM.

**Architecture & Implementation Steps:**

**1. Project Setup & Dependency Management**
*   Initialize the project using `uv` (e.g., `uv init`).
*   Add the following dependencies: `mcp`, `httpx` (for async requests), `trafilatura` (for text extraction), `rank_bm25` (or `meilisearch` if opting for an external vector/text database), and `beautifulsoup4` (for additional HTML parsing if needed).

**2. SearXNG Integration (The Discovery Layer)**
*   Assume the user has a local SearXNG instance running (e.g., at `http://localhost:8888`) with the JSON format enabled in `settings.yml` (`search: formats: - json`).
*   Write an async function to query the SearXNG `/search` endpoint using the `format=json` parameter.
*   Extract the `url`, `title`, `content` (snippet), and any available backlinks/metadata from the JSON response.

**3. Web Scraping with Trafilatura (The Extraction Layer)**
*   Write an async scraping function that takes the list of URLs retrieved from SearXNG.
*   **Ethical Scraping Compliance:** The scraper *must* use a custom, descriptive `User-Agent` string (do not use default library strings). It must implement exponential backoff and rate limiting (e.g., `asyncio.sleep()`) to avoid overloading target servers, and ideally respect `robots.txt`.
*   Use `trafilatura.fetch_url()` and `trafilatura.extract()` to download and extract clean, structured text from the HTML, stripping away ads, navigation, and boilerplate.

**4. Indexing & Ranking (The Relevancy Layer)**
*   Create an in-memory ranking mechanism (such as the BM25 algorithm using the `rank_bm25` library) to score the scraped documents against the original LLM search query. 
*   *Alternative:* If integrating Meilisearch, push the Trafilatura-cleaned documents into a local Meilisearch index, then query it using semantic/hybrid search to retrieve the top chunks.
*   Filter out any documents with a low relevance score to preserve the LLM's context window.

**5. MCP Server Initialization & Tool Definition**
*   Instantiate the server using `mcp = FastMCP("DeepResearchServer")`.
*   Create a tool using the `@mcp.tool()` decorator named `deep_web_research`. 
*   Define a strict Pydantic `BaseModel` or TypeDict for the tool's return schema, returning a list of ranked dictionaries containing: `rank_score`, `url`, `title`, `clean_content`, and `metadata`.
*   **Crucial Rule:** Ensure that all internal logging uses standard error (`sys.stderr`) or the FastMCP `Context` logger. **Never print to `stdout`**, as this will corrupt the JSON-RPC STDIO transport required by the MCP protocol.

**6. Execution**
*   Include the standard `if __name__ == "__main__": mcp.run()` block to start the server.

### Step-by-Step Test Instructions for Codex

Once the code is generated, use the following step-by-step test plan to verify that the MCP server works perfectly with the LLM.

**Test 1: Environment & Dependency Verification**
1. Ensure the local SearXNG Docker container is running and accessible at `http://localhost:8888`. Verify JSON is enabled by running: 
   `curl -kLX GET "http://localhost:8888/search?q=test&format=json"`.
2. Run `uv sync` to ensure `mcp`, `trafilatura`, and ranking libraries are installed.

**Test 2: Server Startup & STDIO Integrity**
1. Run the server script manually via the terminal: `uv run server.py`.
2. The terminal should appear to hang (waiting for JSON-RPC input). 
3. *Pass Condition:* No text or startup logs should print to `stdout`. Any initialization logs you configured must appear in `stderr`. 

**Test 3: MCP Inspector Tool Discovery**
1. Kill the manual process. Use the official MCP Inspector to test the server structure by running:
   `npx @modelcontextprotocol/inspector uv run server.py`.
2. Open the provided `localhost` URL in your browser.
3. Navigate to the "Tools" tab.
4. *Pass Condition:* The `deep_web_research` tool should be listed, displaying the correct input schema (e.g., a `query` string) generated by FastMCP.

**Test 4: End-to-End Execution via Inspector**
1. In the MCP Inspector, select the `deep_web_research` tool.
2. Input a test query, such as `"Recent breakthroughs in solid-state batteries"`, and click "Run Tool".
3. *Pass Condition:* 
   * The tool should not instantly fail (monitor the terminal for `stderr` logs showing SearXNG fetching and Trafilatura scraping).
   * After a few seconds, the UI should display a structured JSON response.
   * The response must contain cleanly extracted text (no raw HTML tags) and show a calculated ranking score for each URL.
   * The results must be sorted by the highest ranking score first.

**Test 5: Integration with Codex / LLM Host**
1. To connect this to Codex, edit your Codex configuration file (usually located at `~/.codex/config.toml`).
2. Add the server configuration under the `mcp_servers` table:
   ```toml
   [mcp_servers.deep_research]
   command = "uv"
   args = ["run", "/absolute/path/to/your/server.py"]
   ```
3. Save the file and open your Codex CLI or IDE extension.
4. Type `/mcp` in the Codex terminal to verify the server is active.
5. Prompt Codex: *"Please use your deep research tool to find out what the current state of the art is for local LLM context windows."*
6. *Pass Condition:* Codex should successfully invoke the tool, wait for the scraping and ranking to complete, read the injected high-quality context, and synthesize a final conversational answer citing the scraped URLs.
